# 生猪体重评估模型算力预估与 3 天实施计划

## 1. 算力需求预估

基于 PIGRGB-Weight 数据集（约 13,000 张图像）和 ResNet-18 模型架构，以下是训练所需的算力资源预估：

### 资源配置建议
| 资源类型 | 推荐配置 | 说明 |
| :--- | :--- | :--- |
| **GPU** | NVIDIA RTX 3060 (12GB) 或更高 | 显存需大于 8GB 以支持 Batch Size 32+ |
| **CPU** | 8 核以上 | 用于数据预处理和增强 |
| **内存** | 16GB RAM | 确保数据加载流畅 |
| **存储** | 50GB SSD | 用于存放数据集和模型 Checkpoints |

### 训练时间预估
*   **单次训练 (50 Epochs)**: 在单张 RTX 3060 上预计耗时 **4-6 小时**。
*   **超参数搜索**: 若进行 3-5 组实验，总计需 **20-30 小时**。

---

## 2. 三天完成计划 (3-Day Sprint)

为了在三天内交付可用的模型，我们将采取并行化策略：

### 第一天：环境准备与数据处理
*   **上午**: 算力资源到位，配置深度学习环境（CUDA, PyTorch）。
*   **下午**: 下载 PIGRGB-Weight 数据集，编写数据清洗脚本，将文件名中的体重信息提取为 CSV 标签文件。
*   **晚上**: 运行初步的数据分布分析，确保训练集和验证集的体重分布均匀。

### 第二天：模型训练与优化
*   **上午**: 启动基准模型（ResNet-18）训练，记录 Loss 曲线。
*   **下午**: 引入数据增强（旋转、翻转、亮度调节）以提高模型泛化能力。
*   **晚上**: 尝试更深的模型（如 ResNet-50 或 EfficientNet-B0）进行对比实验。

### 第三天：评估、部署与交付
*   **上午**: 在独立测试集上评估模型精度（计算 MAE, RMSE, R²）。
*   **下午**: 编写推理脚本，支持单张图片和视频流的实时体重预测。
*   **晚上**: 整理项目文档，上传最终模型权重，完成交付。

---

## 3. 关键里程碑
1.  **Day 1 End**: 数据集就绪，首个训练脚本在 GPU 上跑通。
2.  **Day 2 End**: 获得 RMSE < 5kg 的优化模型。
3.  **Day 3 End**: 交付包含推理接口的完整 GitHub 仓库。
